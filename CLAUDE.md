This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## プロジェクト概要

**AI事始(AIことはじめ)** — 医療者のための生成AI自習サイト。医療系教育者が「リンクを送るだけ」で、学習者(研修医・医学生等)が生成AIの使い方を自習し、AIフィードバックを受け、修了証(「目録」)を自己発行できる静的Webサイトを作るプロジェクト。名称の由来は杉田玄白『蘭学事始』。

## 現在の状態(重要)

**企画フェーズ。コードはまだ存在しない。** リポジトリは git 未初期化で、`documents/` 配下の2つの正本ドキュメントのみがある:

- `documents/AI事始-要件定義書-v0.6.md` — 要件・技術構成・MVP定義・未決事項
- `documents/AI事始-初級コンピテンシー-v0.2.md` — 学習目標(A/B/C領域)とユニット構成(確定版)

ビルド・テストコマンドは存在しない。実装開始時の予定スタック: **Material for MkDocs + GitHub Pages**、クイズと修了証生成は静的JS(Canvas/jsPDF)でブラウザ内完結。

## 設計原則(全実装を拘束する制約)

要件定義書 §1.3 より。実装・提案はすべてこれに従うこと:

1. **教員の手間を増やさない** — 教員の仕事は「リンクを送る」のみ
2. 学習者は体験〜フィードバック〜修了証発行まで一人で完結
3. 保守は運営者一人が **Markdown編集だけ** で回る(AI指示文の正本もリポジトリ内のmd)
4. **ユーザーのログ・個人情報をサイト側に一切残さない** — サーバ送信ゼロ、アクセス解析も禁止。ログイン・進捗管理・LMS・API連携チャットは非目標(§3)

## アーキテクチャ上の主要決定

- **コンテンツ構成**: 初級3領域×各2ユニット(A: メカニズム / B: 対話スキル / C: 倫理・安全・検証)。B領域は職種別(医師/看護/薬学)タブで題材のみ差し替え、構造・解説・クイズは共通
- **クイズ**: 領域ごと、正答率8割合格、再挑戦無制限。修了(目録発行)の条件はクイズ合格のみ(ユニット閲覧は不問)
- **修了証**: 呼称は「目録」(3領域制覇で「初級目録」)。氏名入力→ブラウザ内でPNG/PDF生成。公式証明ではない旨を明記
- **AIフィードバック**: Gem版(メイン、共有リンク)+ 汎用版プロンプト(フォールバック)の2段構え。**汎用版プロンプトのmdが指示文の正本**であり、Gemへは手動同期(同期日をmdに記録)。指示文は公開前提で秘匿しない
- **陳腐化対策**: 普遍的内容とツール固有How-toを分離して書く
