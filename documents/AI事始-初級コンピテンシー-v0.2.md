## 前提

- 初級修了 = 以下の3領域すべての修了証(目録)を取得した状態
- 修了判定はクイズ合格のみ(4.3の決定事項)。したがって各コンピテンシーはクイズで評価可能な形に落とす
- スキル系コンピテンシーはシナリオ型設問(状況を提示し、次に取るべき行動を選ばせる)で近似評価する
- 修了証の呼称案: 領域修了 =「目録」、初級全領域修了 =「初級目録」(将来、上級で「免許皆伝」)

## 領域A: 生成AIのメカニズム

| # | コンピテンシー | 評価方法の目安 |
|---|---|---|
| A1 | LLM(大規模言語モデル)が「次に来る語の確率的な予測」で文章を生成していることを、自分の言葉で説明できる | 知識問題+誤説明の指摘問題 |
| A2 | 拡散モデル(画像生成)が「ノイズから徐々に画像を復元する」仕組みであり、LLMとは原理が異なることを説明できる | 知識問題(LLMとの対比) |
| A3 | ハルシネーション(もっともらしい誤り)が仕組み上避けられない理由を、A1と関連づけて説明できる | 知識問題+事例判定 |
| A4 | 生成AIがテキスト以外(画像、スライド、コード、音声等)も生成できることを知り、自分の業務での用途を挙げられる | 知識問題 |

## 領域B: 対話的活用スキル

| # | コンピテンシー | 評価方法の目安 |
|---|---|---|
| B1 | プロンプトは一度入れて終わりではなく、出力への追加指示による反復改善が基本であることを説明でき、実践できる | シナリオ型(不十分な出力を提示→次の一手を選択) |
| B2 | コンテキスト(背景、対象者、制約条件)を与えることで出力が改善することを、具体例(例: 授業カリキュラムの作成)で示せる | シナリオ型(コンテキスト有無の出力比較) |
| B3 | 不足情報をAI側から質問させる指示、および忖度のない批評・ダメ出しをさせる指示を使える | シナリオ型+成果物課題(フィードバックAIへの提出) |

## 領域C: 倫理・安全・検証

| # | コンピテンシー | 評価方法の目安 |
|---|---|---|
| C1 | 患者情報・個人情報を生成AIに入力してはいけない理由を説明でき、入力前に匿名化・抽象化する方法を実践できる | シナリオ型(この入力文の問題点はどれか) |
| C2 | 生成物と学習データをめぐる著作権・倫理上の留意点(生成物の利用、既存著作物の入力、出典の確認)を説明できる | 知識問題+事例判定 |
| C3 | 生成AIの出力には学習データ由来のバイアスが必ず含まれることを前提として説明できる | 知識問題 |
| C4 | 生成AIの出力を業務に用いる前に、人間による検証が必須であることを説明でき、検証の観点(事実確認、対象者への適合、有害性)を挙げられる | シナリオ型(この出力のどこを検証すべきか) |

## ユニットへのマッピング(案)

| 領域 | ユニット案 | 対応コンピテンシー |
|---|---|---|
| A | Unit A-1「AIはなぜ間違えるのか: LLMの仕組み10分入門」 | A1, A3 |
| A | Unit A-2「文章だけじゃない: 画像生成の仕組みと使いどころ」 | A2, A4 |
| B | Unit B-1「一発で諦めない: 対話で育てる」(職種別題材) | B1, B2 |
| B | Unit B-2「AIに質問させる・ダメ出しさせる」(職種別題材) | B3 |
| C | Unit C-1「患者情報を守る: 入力前の3秒チェック」 | C1 |
| C | Unit C-2「著作権・バイアス・検証: 使う前の作法」 | C2, C3, C4 |

計6ユニット(各10分以内)+ 領域ごとのクイズ+目録発行。

### B領域の職種別題材(構造共通・お題差し替え方式)

ユニットの構成、解説、クイズは全職種共通とし、お題(プロンプト例)と期待される出力例のみを職種タブで切り替える。制作・保守の増分を最小化する。

| 職種 | B-1題材(案) | B-2題材(案) |
|---|---|---|
| 医師 | 研修医向け勉強会のカリキュラム作成 | 学会発表スライド構成へのダメ出し依頼 |
| 看護 | 新人向け手技説明資料の作成 | 院内研修企画への批評依頼 |
| 薬学 | 患者向け服薬指導資料の作成 | 実習生指導計画への批評依頼 |

### クイズ仕様

- 合格基準: 正答率8割以上(領域ごと)
- 再挑戦: 無制限(静的サイトのため回数制限は設けない。設問プールからのランダム出題は将来検討)
